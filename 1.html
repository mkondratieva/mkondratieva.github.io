В 3 семестре будем изучать C++ и Python (+коллоквиум по Си). Желательноо имееть на домашнем ПК возможность работать в Linux, см. инструкция.

Выкладываю пример достаточно наивной реализации на Си списка списков (=кривой двумерный массив), в котором рекомендуется разобраться перед тем, как переходить на С++.

Пройденный во 2 семестре материал:

Задание 7. Си, структуры. Список задач 7, пример части Б.

Задание 6. Питон, сортировка массива. Список задач 6. Пример программы.

Задание 5. Питон, работа c массивом. Список задач 5.

Задание 4. Питон, обработка последовательности. Список задач 4. 2 варианта просмотреть и распечатать все числа из файла, с тестами, документ-строкой и обработкой ошибок.

Задание 3. Работа с кривыми двумерными массивами, СИ. Список задач 3. Текст с пояснениями относительно задачи 3. Пример с кривым двумерным массивом. Пример проекта: заголовочный и файл с реализацией задачи об обнулении всех элементов двумерного массива.

Задание 2. Работа со строками, Си. Список задач 2. Для знакомства со строками рекомендуется прочитать следующий документ и посмотреть примеры программ 1, 2, 3, 4, и 5. В задачах требуется, вызвав в цикле fgets, просмотреть все строки файла, выделить слова, удовлетворяющие некоторому условию, по заданному критерию выбрать из них одно и распечатать. Слова можно выделять вручную (используя strchr), с помощью функции sscaf или применяя strtok. Строки в файле имеют известное ограничение на размер, поэтому память можно выделять статически (в этом случае не будет ее утечек).

Задание 1. Двумерные массивы известного размера, Си. Список задач 1. В программе должна присутствовать функция, отводящая память под (не одномерный!) двумерный массив и заполняющая его числами из файла. Ошибки обработать. Количество чисел в файле должно соответствовать заявленным размерам (иначе -- выдача сообщения). При обработке ошибок можно не отслеживать количество чисел в строке, а только их общее количество в файле. Файл открываем один раз. Вся захваченная память должна корректно освобождаться. Преобразование массива (удаление строк или столбцов) должно быть в отдельной функции. Программу разделить по 3 файлам: заголовочный, с функцией main и реализация всего остального. Печать можно делать на экран. Вариант без обработки ошибок. Еще один пример отводит память под матрицу единым куском.

Инструкция по установке компилятора (для OS Win). Для тех, у кого есть компилятор: читайте документ с конца (пункт, выделенный красным). Собирать проект под Linux нужно командой make, если у вас MinGW, то mingw32-make (или ...64..). Make файл позволяет не только собирать весь проект одной командой, но и делать это оптимально. Т.е. если вы внесли изменения только в один файл, лишние команды не запускаются. Принцип написания Makefile достаточно прост. Он состоит из целей, зависимостей и команд. Команды перечисляются каждая с новой строки. Перед каждой командой используем табуляцию. Связка цель: зависимость может иметь вид: file.o:file.c file.h (строка без отступов, перед одной или несколькими командами). Смысл связки: если вы что-то изменили в проекте (определяется по дате цели и зависимости, либо по отсутствию цели), то блок команд выполняется. Пример Makefile (для проекта, состоящего из заголовочного файла matrix.h и двух файлов с реализациями, matrixget.c и task.c, в которых есть включение заголовочного #includ"matrix.h". Не включайте текст одного .c файла в другой!).
