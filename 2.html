Работа на ЭВМ, 2 курс, механики, 2025-2026 г.

Распределение задач.

25.05 можно переписать одну из несданных контрольных работ.

Задания принимаются только в виде проекта. Используем при С++ компиляции ключ:

-fno-elide-constructors -std=c++11

Задача 1. Срок сдачи 1:2.03 (включительно).

В задаче 2 нужно сделать то же, что и в 1, но с использованием STL (выбрать подходящий контейнер, использовать string для имени файла и извлечения данных, массив умных указателей вмеcто обычных, без new и delete). Срок сдачи задачи 2: 16.03.

В задаче 3 нужно сделать то же, что и в 1, 2 + распараллелить циклы (использовать инструкции OpenMP) + сравнить время работы (в 1 и 2 задаче). Срок сдачи задачи 3: 6.04.

Питон, задача 4. Задачи на графику требуют установить Питон 3 и, возможно, пакеты. Для работы с изображениями в Win используем imageio, в Linux -- misc из пакета scipy (misc.imread, misc.imsave). К этой задаче нужно написать отчет в TEX (отчет можно сдать позднее, до 11 мая). Скачать MiKTex можно на официальном сайте. Срок сдачи задачи 4: 20.04 (включительно).

Задача 5, Питон, работа с bmp-изображениями. Срок сдачи: 18.05. Решение предполагает использование интерполяции функции нескольких переменных и (или) векторизации растровых изображений.

9.02.С++. Наследование. Полиморфные классы. Конструкторы и десрукторы в базовых и дочерних классах, final, vortual. Статический и динамический полиморфизм. Срезка. Ссылки и указатели на базовый класс (на дочерний класс). Набросок написанного на занятии кода.

16.02.С++. Наследование. Полиморфные классы. Таблица виртуальных функций. Исключения. Написанный на занятии код.

02.03.С++. Наследование. Абстрактный класс. Минимальная сигнатура для решения домашней задачи. Фабрики.

16.03.С++. Наследование. STL. Тест на работу move-конструктора в классе, содержащем string.

30.03. OpenMP, некоторые возможности. При компиляции нужно использовать ключ -fopenmp. Примеры с корректным определением времени работы в многопоточном приложении. Полезные функции.

6.04. Питон, графика. Входной файл можно создать в Paint (Win), сохранив в 24-разрядном bmp-формате, или аналогично, в программе KolorPaint (Linux). Пример обработки изображения.

20.04. Питон, графика. Пример отчета в тех. Разбор варианта контрольной работы, набросок кода.

4.05. Питон, графика. Интерполяция функции двух переменных. Метод обратных расстояний (см. презентацию Староверова). Разбор вариантов контрольной работы 3.

11.05. Питон, графика. Векторизация. eps- файлы.

Чтобы в Linux скачать с github публичный репозиторий (пароль не требуется): git clone https://github.com/mkondratieva/2kurs.git

Пакетные файлы (жесткие ключи) wgcc, wg++ для Win (.bat) и Linux (без расширения). Для установки (Win) после распаковки положите файлы в директорию, где находится gcc.exe. Инструкцию по установки для Linux можно найти здесь (строки 62-77). Для проверки утечек памяти (Win) файл malloc.cpp необходимо вставить в ваш проект.

Если дома вы работаете не в Linux, по возможности надо установить вторую ОС. Как можно установить две операционные системы, советы: Win, Mac . По последней ссылке мой опыт работы на Мас и инструкции по установке гостевой Ubuntu в Virtual Box (для любой ОС).
