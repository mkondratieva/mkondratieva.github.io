Питон, графика, задача 4. Срок сдачи -- 7 мая.

Требования к дистанционной сдаче C++ программ: программа должна быть поделена на файлы, к проекту требуется приложить Malefile. О том, как его написать, можно посмотреть на странице для 1 курса, там же есть инструкции по установке компилятора и работе в командной строке (Win). Работать со средой даже дома нельзя (это требование лектора)! Пример второй задачи. Вместо графа взяла вектор, в выводе используется строка string с именем файла. Файл для тестирования. Базовый класс можно объявить наследником двух классов, один -- тот, что подходит для задачи, другой string. Классы тогда получаются простые, поэтому перегрузки операторов минимальные (в примере их даже больше, чем нужно в задачах). По поводу первой задачи: версия 3 -- практически готовый пример.

Первая задача, тема наследование (С++). Требования аналогичны старым. Программа должна быть разделена по файлам, не заканчивать работу аварийно, писать понятные результаты. Не использовать StL, другие библиотеки требуют согласования. При считывании данных из файла нельзя дважды открывать файл и возвращаться к его началу.. Количество строк и их длина заранее неизвестны. Компилируем с ключом, отключающим оптимизацию:

g++ -fno-elide-constructors -std=c++11

Вторая задача -- все то же, что и в первой, но с использованием STL. В качестве данных базового класса взять подходящий контейнер STL, в циклах использовать только итераторы. В обеих задачах должны корректно работать конструкторы (из одного дочернего класса в тот же и в другой, по указателю на базовый класс, а если это возможно, то и конструктор перемещения), присваивания (в том числе объекта одного дочернего класса объекту другого дочернего класса и через указатель на базовый класс, а если возможно, то и присваивание перемещением), и суммы объектов различных дочерних классов, в том числе через указатель на базовый класс. Срок сдачи обеих задач -- 26 марта.

Задача 3, тема параллельные вычисления. Срок 23 апреля включительно. Используем механизм OpenMP, справку можно найти на сайте лектора и, например, тут. Для сборки проекта компилятору добавить ключ -fopenmp. Требуется распараллелить циклы в задачах 1 и 2 и сравнить результаты и время работы, варианты с параллельными вычислениями должны работать быстрее. Для тестирования взять массив указателей большого размера, построить его считыванием из файла и многократным копированием полученных объектов. Или, как вариант, взять объекты большого размера, полученные копированием (или суммированием) данных исходных объектов (прочитанных из входного файла). Прислать скрин результатов или продемонстрировать работу программы в Zoom. Пример правильного и неправильного распараллеливания тут.

Пример программы, в которой объявляется базовый и дочерние классы с виртуальными функциями. Далее с помощью массива фабрик, создающих указатели на дочерний и базовый классы строится массив указателей на базовый класс.

На семинарах разбираем пример, аналогичный домашней задаче. В домашней задаче также должны быть перегружены конструкторы копирования и перемещения, присваивания копированием и присваивание перемещением. 05.03 написали функции для извлечения строки неограниченного размера и чисел из строки: инструменты для работы с файлом. Задача о графе была продвинута до версии 3. Это близко к финальной, осталось только в build дописать случай неограниченного количества строк.
